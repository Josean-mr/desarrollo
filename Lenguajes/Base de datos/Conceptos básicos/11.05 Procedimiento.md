# ¿Qué es?
- Un procedimiento es un programa que ejecuta una acción específica y que no devuelve ningún valor.
- Un procedimiento tiene un nombre, un conjunto de parámetros (opcional) y un bloque de código.
<font style="color:salmon; font-weight:bold">Sintaxis:</font>
```sql
CREATE [OR REPLACE] PROCEDURE <proc_mame> [(<param1>)] [IN|OUT|INT OUT] <type>, <param2> [IN|OUT|IN OUT] <type>, ...)]
AS -- declaración de variable locales
BEGIN -- Sentencias
[EXCEPTION] -- Sentencias control de excepción
END [<procedure_name>];
```
## Parámetros
- Se debe especificar el tipo de datos de cada parámetro.
- No se debe especificar la longitud del tipo.
- Los parámetros pueden ser:
	- De entrada: **IN** -- valor por defecto.
	- De salida: **OUT**.
	- De entrada salida: **IN OUT**.
<font style="color:green; font-weight:bold">Ejemplo:</font>
```sql
CREATE OR REPLACE PROCEDURE Actualiza_Saldo(cuenta IN NUMBER, new_saldo OUT NUMBER)
AS
  -- declaración de variables locales
BEGIN
UPDATE SALDOS_CUENTAS
SET SALDO = new_saldo
WHERE CO_CUENTA = cuenta;
END Actualiza_Saldo;
```
### Parámetros <font style="color:#08F">Notación posicional</font> y <font style="color:#08F">notación nominal</font>
<font style="color:#08F; font-weight:bold">Notación posicional:</font>
- Se pasan los valores de los parámetros en el mismo orden en que el PRODCEDURE los define.
```
BEGIN
	Actualiza_Saldo(200501, 2500);
	COMMIT;
END ;
```
<font style="color:#08F; font-weight:bold">Notación nominal:</font>
- Se pasan los valores en cualquier orden nombrando explícitamente el parámetro.
```
BEGIN
	Actualiza_Saldo(cuenta => 200501, new_saldo => 2500);
	COMMIT;
END ;
```
# Ejecutar procedimientos
- Dos maneras:
	- Haciendo uso del nombre del procedimiento dentro de un bloque PL/SQL:
```sql
BEGIN
	Actualiza_Saldo (200501, 2500);
END;
```
- Mediante la sentencia EXEC de la siguiente manera:
```sql
exec Actualiza_Saldo (200501, 2500);
```
# Borrar procedimiento almacenado
<font style="color:salmon; font-weight:bold">Sintaxis:</font>
```sql
DROP PROCEDURE [IF EXISTS] <proc_name>
```
