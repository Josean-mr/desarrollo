La programación orientada a objetos (POO) es un paradigma de programación que organiza el código en "objetos," que son instancias de clases. Estos objetos interactúan entre sí para realizar tareas y resolver problemas. Cada objeto contiene datos (atributos) y comportamientos (métodos), y la POO se enfoca en la creación y manipulación de estos objetos para desarrollar software de manera modular, reutilizable y más fácil de mantener.

# Principios de la programación orientada a objetos
[Link](https://profile.es/blog/que-es-la-programacion-orientada-a-objetos/)
## <font style ="color:FireBrick">Encapsulación</font>
Contiene la información importante de un objeto dentro del mismo y solo expone al exterior la información seleccionada (ver modificadores de acceso en [[07.3_Métodos]])
## <font style ="color:FireBrick">Abstracción</font>
Es cuando el usuario interactúa solo con los atributos y métodos seleccionados de un objeto.
## <font style ="color:FireBrick">Herencia</font>
Define relaciones jerárquicas entre clases.
## <font style ="color:FireBrick">Polimorfismo</font>
Consiste en diseñar para compartir comportamientos.
# ¿Qué es un objeto?
La materialización de una clase. El objeto de una representación de dicha clase.

## Ventajas de los objetos
- Reutilización del código.
- Más sencillo de añadir cambios, modificaciones, etc.
- Los problemas orientados a objetos son más fáciles de resolver.
- Aislamiento de errores en los problemas
# <font style = "color:#7dcea0">Operador new</font>
## ¿Para qué sirve?
Para crear una instancia de un tipo de objeto definido por el usuario o de unos de los tipos de objeto integrados que tiene una función constructora.
