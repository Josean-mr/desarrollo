## Tipos de método
En java solo se habla de métodos, pero existen 3 tipos de métodos:
- **Procedimiento**
	- No devuelve nada.
	- void.
![[Pasted image 20240901165835.png]]
- **Método**
	- Tiene un tipo de retorno.
	- Recibe parámetros (int num1, int num2).
![[Pasted image 20240901170445.png]]
- **Función**
	- Tiene retorno.
	- No recibe parámetros.

### Parámetros vs argumentos
#### Parámetros
Son las **variables o datos que se declaran al crear un método**. Es lo que hay entre paréntesis.
	public double sumar2Numeros (int num1, int num2) {
	double resultado = 0;
	resultado = num1 + num2;
	return resultado;
	}
- En el ejemplo, **los parámetros son las variables num1 y num2**.
#### Argumentos
Son **valores o variables que se pasan para llamar al método**.
	sumar2Numeros(1, 3);
- En el ejemplo, **los argumentos son el 1 y el 3**.
# <font style = "color:orange">return</font>
## ¿Qué es?
Es una sentencia que se emplea para salir de la secuencia de ejecución de las sentencias de un método y, opcionalmente, devolver un valor.
## Uso
### Sola
Para salir del metodo;

### Con una expresión
Para salir y devolver un valor.

#### Detrás de return no se ejecutará nada más
# <font style = "color:orange">this</font>
[Link](https://gl-epn-programacion-ii.blogspot.com/2010/03/setters-getters.html)
## ¿Qué es?
Es una variable de referencia que hace referencia al objeto actual. Básicamente se utiliza para distinguir los atributos (variables) de la clase, con las variables de un método.
## Getter
Es un método que se encarga de <font style = "color:red">devolver</font> un valor de un atributo de una clase.
## Setter
Es un método que se encarga de <font style = "color:red">asignar</font> un valor recibido por parámetro a un atributo de una clase. Setter no retorna nada(es void).
