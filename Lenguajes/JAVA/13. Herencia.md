# ¿Qué es?
- La herencia es un mecanismo que permite la definición de una clase a partir de la definición de otra ya existente.
- La herencia permite heredar a las clases características y conductas de una o varias clases denominadas base.
<font style="color:green; font-weight:bold">Ejemplo:</font>
```java
package herencia.unica;

// Definir clase Padre
public class Padre {

	// Atributos
	protected String nombre;
	protected String apellido;

	// Constructor de la clase Padre
	public Padre(String nombre, String apellido) {
		this.nombre = nombre;
		this.apellido = apellido;
	}

	public String getNombre() {
		return nombre;
	}

	public void setNombre(String nombre) {
		this.nombre = nombre;
	}

	public String getApellido() {
		return apellido;
	}

	public void setApellido(String apellido) {
		this.apellido = apellido;
	}

}

// Definir clase Hija que heredará variables y métodos de la clase Padre
class Hija extends Padre {

	// Atributos
	private int edad;

	/* 
	* Constructor de la clase hija con su parámetro y
	* los parámetros heredados
	*/
	public Hija(String nombre, String apellido, int edad) {

		/*
		 * Uso de la palabra reservada super para indicar que se
		 * tratan de variables heredadas
		 */
		super(nombre, apellido);
		this.edad = edad;
	}

	public int getEdad() {
		return edad;
	}

	public void setEdad(int edad) {
		this.edad = edad;
	}
	
	// Método toString que imprimirá los datos cuando se invoque
	@Override
	public String toString() {
		return "Nombre: " + nombre + "\nApellidos: " + apellido + "\nEdad: " + edad;
}

class Main {
	public static void main(String[] args) {
		/* 
		 * Creación de objeto de la clase Hija que contiene atributos
		 * que hereda de la clase padre
		 */
		Hija hija = new Hija("Tere", "García", 23);
		
		// Invocación del método toString
		System.out.println(hija.toString());
		}
	}
}
```

# Jerarquía de la herencia
## Constructores sin parámetros
- El orden de la jerarquía de la ejecución de los constructores [[07.3. Constructores]] es de arriba a abajo.
<font style="color:green; font-weight:bold">Ejemplo:</font>
```java
package herencia;

// Super clase
public class A {
	A() {
		System.out.println("En A");
	}
}

// Subclase de A
class B extends A {
	B() {
		System.out.println("En B");
	}
}

// Subclase de B
class C extends B {
	C() {
		System.out.println("En C");
	}
}

// Clase con método Main para mostrar la jerarquía de la herencia
class Constructores_y_Herencia {
	public static void main(String[] args) {
		C obj = new C();
	}
}
```

## Constructores con y sin parámetros (sobrecarga del constructor)
- Cuando se crea un objeto de una subclase con parámetros, no se ejecutara el constructor con parámetros de la super clase, ya que no se invoca directamente el constructor de la superclase y por defecto se ejecutará el constructor sin parámetros.
<font style="color:green; font-weight:bold">Ejemplo:</font>
```java
package herencia.sobrecarga.constructor;

// Super clase
public class A {
	A() {
		System.out.println("En A");
	}
	
	A(int i) {
		System.out.println("En A(i)");
	}
}

// Clase hija de la clase A
class B extends A{
	B() {
		System.out.println("En B");
	}
	
	B(int j) {
		System.out.println("En B(j)");
	}
}

// Clase con método main
class Herencia_y_Sobrecarga_Constructores {
	public static void main(String[] args) {
		B obj = new B(5);
	}
}
```
